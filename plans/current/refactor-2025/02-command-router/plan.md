# План этапа 02: CommandRouter и CommandHandler'ы

## Цель
- Вынести обработку команд из MessageHandler в отдельный CommandRouter
- Создать отдельные CommandHandler'ы для каждой команды
- Уже частично реализовано: команды вынесены, но нет централизованного CommandRouter

## Шаги
1. Создать интерфейс IBotCommandHandler
2. Реализовать CommandRouter с маршрутизацией по имени команды
3. Вынести существующие команды в отдельные классы-обработчики
4. Зарегистрировать все CommandHandler'ы в DI
5. MessageHandler должен делегировать команды через CommandRouter
6. Протестировать работу команд
7. Описать изменения в ворклоге

## Ветка
- `refactor/command-router`

## Критерии готовности
- MessageHandler не содержит логику команд
- Все команды обрабатываются через CommandRouter
- Все тесты проходят
