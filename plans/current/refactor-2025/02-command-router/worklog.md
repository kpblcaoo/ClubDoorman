## Изменения в архитектуре

### Новые компоненты:
1. **ICommandRouter** - интерфейс для маршрутизации команд
2. **CommandRouter** - реализация маршрутизатора, использует Dictionary для быстрого поиска
3. **Отдельные Command Handlers** - каждая команда теперь имеет свой класс

### Принцип работы:
1. MessageHandler.HandleCommandAsync() сначала вызывает CommandRouter.HandleCommandAsync()
2. CommandRouter ищет подходящий обработчик по имени команды и вызывает его
3. Если CommandRouter не нашел обработчик, MessageHandler использует fallback к старым методам
4. Это обеспечивает 100% обратную совместимость

### Регистрация в DI:
- Все новые Command Handlers зарегистрированы как ICommandHandler
- CommandRouter зарегистрирован как ICommandRouter
- Старые интерфейсы (IStartCommandHandler, ISuspiciousCommandHandler) сохранены для совместимости

### Логика команд:
- Вся логика скопирована дословно из MessageHandler без изменений
- Проверки прав администратора сохранены
- Проверки админ-чатов сохранены
- Все уведомления и обработка ошибок сохранены
