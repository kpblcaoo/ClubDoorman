# Общий план рефакторинга ClubDoorman (2025)

## Цели
- Повысить поддерживаемость и тестируемость кода
- Упростить DI-граф, убрать сервис-локатор
- Разгрузить MessageHandler и повысить читаемость
- Ускорить работу с кэшем пользователей
- Улучшить трассировку логов

## Этапы

1. **Явные зависимости**
   - Убрать IServiceProvider из handler'ов и сервисов, все зависимости через конструктор
   - Проверить DI-регистрации на корректность
   - Уже частично реализовано: StartCommandHandler, SuspiciousCommandHandler, ILogChatService — но вызываются через локатор

2. **CommandRouter**
   - Вынести обработку команд в отдельный CommandRouter
   - Создать отдельные CommandHandler'ы для каждой команды
   - DI-регистрация всех команд
   - Уже частично реализовано: команды вынесены, но нет централизованного CommandRouter

3. **Handler Slimming**
   - Вынести AI, уведомления, пересылки, кнопки и т.д. в отдельные сервисы
   - MessageHandler оставить только маршрутизацию

4. **UserIndex**
   - Ввести отдельный индекс username → userId
   - Убрать O(n) обходы MemoryCache

5. **Логирование с scope**
   - Внедрить BeginScope с ChatId/UserId/MsgId/TraceId

6. **Тесты и документация**
   - Покрыть изменения тестами
   - Описать изменения в документации

## Порядок внедрения
- Каждый этап — отдельная ветка и рабочая папка
- После завершения этапа — ревью, тесты, слияние
- Минимизировать параллельные изменения

## Уже реализовано/заделы
- StartCommandHandler, SuspiciousCommandHandler, ILogChatService — есть, но требуют доработки для чистой архитектуры
- Остальные сервисы и паттерны — реализовать с нуля
