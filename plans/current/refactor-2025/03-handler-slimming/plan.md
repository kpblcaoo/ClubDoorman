# План этапа 03: Handler Slimming (разгрузка handler'ов)

## Цель
- Вынести AI, уведомления, пересылки, кнопки и т.д. из MessageHandler в отдельные сервисы
- MessageHandler оставить только маршрутизацию и базовую обработку

## Шаги
1. Проанализировать обязанности MessageHandler
2. Вынести AI-каскад, уведомления, пересылки, кнопки в отдельные сервисы
3. Внедрить новые сервисы через DI
4. MessageHandler делегирует соответствующие задачи
5. Протестировать работу бота и тестов
6. Описать изменения в ворклоге

## Ветка
- `refactor/handler-slimming`

## Критерии готовности
- MessageHandler содержит только маршрутизацию и базовую обработку
- Вся логика вынесена в отдельные сервисы
- Все тесты проходят
