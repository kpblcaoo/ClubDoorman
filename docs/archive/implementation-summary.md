## 🚀 Как это работает

1. **Проверка настройки** - функция проверяет, включена ли в конфигурации
2. **Анализ события** - проверяет, что пользователь действительно вошел через папку (`via_chat_folder_invite_link = true`)
3. **Автоматический бан** - вызывает `AutoBanAsync` с причиной "Автоматический бан за вход через папку"
4. **Логирование** - записывает событие в логи
5. **Уведомление** - отправляет уведомление администратору (через существующую систему)

### Phase 4 (Schema=4) Детали
Цель: предоставить «самый тонкий» стабильный слой для будущего сравнения без шума. Поля: `Schema`, `Id`, `CorrelationId`, `ShortName`, `Action`, `RuleCode`.

Источник данных:
1. Первично — V2 (`baseline_v2/*.v2.json`).
2. (Удалено) Fallback — исходный v1 `*.output.json`.
2*. Новый fallback: локальный `*.sem.json` (не коммитится).

Инварианты проверяются тестом `GoldenNormalizationTests` (Category=GoldenNorm):
- Наличие файла для каждого manifest entry.
- `Schema == 4`.
- Совпадение `Id/CorrelationId/ShortName` и (если не null) `Action` / `RuleCode` с manifest.
- Полное множество `RuleCode` идентично manifest.

Назначение слоя Schema=4:
- Будущая точка входа для дифф-валидации PR (CI gate) с минимальными ложными диффами.
- (Сделано Phase 9) Убраны громоздкие v1 `*.output.json`.

След. шаги (план):
1. Phase 9 — окончательное удаление v1; введён локальный semantics слой (`*.sem.json`).
2. Phase 8 — Mutation/chaos проверки стабильности (опционально) + hash цепочка для quick integrity check.

### Phase 6 (CI Gate) Детали

Цель: мгновенно подсветить любые непреднамеренные изменения семантики модерации. Если изменение осознанное — разработчик обновляет golden baseline локально и коммитит.

## 🔄 Channel Effects Migration Progress (Stage 2 & 3) — HISTORICAL (migration completed, scaffolding removed)

Дата (историческая фиксация этапа): 2025-08-17

### Цель (исторически)
Поэтапный переход от «legacy» модерации каналов к новой effects-пайплайн архитектуре с режимом dual-run для сравнения результатов перед окончательным переключением. ЭТА ФАЗА ЗАВЕРШЕНА: legacy путь, dual-run и флаги переключения удалены.

### Что было сделано (на том этапе)
1. Добавлены конструкторные override-параметры в `ChannelModerationService`:
	- `overrideEffectsEnabled`
	- `overrideDualRunEnabled`
	- `overrideChannelAutoBan`
	Это позволяет тестам детерминированно активировать/деактивировать флаги, обходя статические env-based значения в `Config`.
2. Реализованы и стабилизированы юнит-тесты `ChannelEffectsPipelineTests` (4 сценария) — (УДАЛЕНО после завершения миграции как временное scaffolding):
	- Legacy only (effects off)
	- Effects only (dual-run off)
	- Dual-run parity (оба пути выполняются, результат совпадает)
		- Dual-run mismatch (разные действия — теперь утверждается наличие warning-лога с шаблоном и значениями Legacy/Effects через in-memory `TestLogger`)
3. Исправлена проблема: «autoban» ветка (по умолчанию) преждевременно завершала обработку и скрывала pipeline — решено через `overrideChannelAutoBan=false` в тестах.
4. Убрана хрупкая проверка логов через `Moq` (сложный generic predicate). Запланирована замена на кастомный in-memory logger sink с последующим assert по шаблону.

### Результат тестов (на момент существования dual-run)
Все 4 сценария проходили (NUnit, .NET 9, Debug). Добавлялась строгая валидация warning-лога mismatch через `TestLogger<T>`. После финального удаления legacy пути эти тесты утратили ценность и были удалены.

### Технические заметки (актуализировано)
* Исторические статические флаги в `Config` (effects enabled / dual-run / disable autoban) удалены. Осталась только действующая логика с effects-пайплайном.
* Каноническое поведение теперь определяется исключительно через эффекты; дополнительных snapshot-проверок не требуется.

### Следующие шаги (устарели — заменены итоговым состоянием)
Исторические пункты больше не актуальны: dual-run отключён и удалён, флаги больше не существуют, чеклист выполнен. НОВОЕ состояние: поддерживать эффекты как единственный путь и развивать тесты сервиса без временных абстракций.

### Вывод (итог)
Dual-run и legacy путь полностью выведены из кода. Effects-пайплайн — единственный механизм. Временные тесты и инфраструктурные флаги удалены для упрощения сопровождения.
